# 狼人杀网页版 - 页面设计（手机优先 320–767）

## 0. 设计总则（Mobile-first）
- **主要目标视口**：320–767px；默认按手机布局输出与实现。
- **渐进增强**：
  - 768–1023px：内容居中、卡片栅格变为双列（如房间列表/回放列表）。
  - ≥1024px：最大内容宽度 960–1120px；信息密度提高（左右分栏仅在大屏启用）。
- **布局策略**：移动端以“单列堆叠 + 底部固定主操作（CTA）/顶部吸顶信息条”为主。

## 1. Global Styles（全局样式与设计 Tokens）
### 1.1 颜色
- `--bg`: #0B1020（深色背景）
- `--surface`: #121A2F（卡片/面板）
- `--text`: #EAF0FF（主文本）
- `--muted`: rgba(234,240,255,0.7)（次文本）
- `--primary`: #6C5CE7（主按钮）
- `--danger`: #FF4D4F（危险操作/淘汰提示）
- `--border`: rgba(234,240,255,0.12)

### 1.2 排版（移动端优先）
- 字体：系统字体栈（iOS/Android/Windows 默认）
- 字号：
  - `H1 20/28`（页面标题）
  - `H2 16/24`（区块标题）
  - `Body 14/22`（正文）
  - `Meta 12/18`（辅助信息）

### 1.3 组件基准
- 圆角：卡片 `12px`；按钮 `10px`
- 间距：页面左右内边距 `16px`；区块间距 `12–16px`
- 按钮：
  - Primary：实心主色，禁用态降低不透明度
  - Secondary：描边/半透明底
- 交互与状态：
  - 触控命中区≥44px
  - Loading：按钮内旋转图标 + 文案变为“处理中”
  - 错误：表单项下方红色说明 + 顶部轻提示（Toast）

## 2. 页面：登录/注册页（/auth）
### 2.1 Meta Information
- Title：登录 / 注册 - 狼人杀
- Description：登录后创建或加入房间开始对局。
- OG：`og:title`, `og:description`, `og:type=website`

### 2.2 Layout
- Mobile：单列居中；容器宽度 100%，左右 16px。
- Tablet/Desktop：表单居中卡片，最大宽度 420px。

### 2.3 Page Structure
1. 顶部区：产品名 + 简短 Slogan
2. 表单卡片：登录/注册切换（Tabs）
3. 底部区：协议/隐私链接（可简化为占位文字）

### 2.4 Sections & Components
- Logo/Title
  - 文本 Logo（无需图片）
- Tabs：登录 / 注册
  - 选中态下划线 + 主色
- 表单字段
  - 注册：邮箱/用户名、昵称、密码
  - 登录：邮箱/用户名、密码
  - 字段错误提示：紧贴字段下方
- 主按钮（Primary）
  - “登录”或“注册并登录”
- 次按钮（Secondary）
  - “去注册/去登录”

## 3. 页面：游戏大厅页（/lobby）
### 3.1 Meta Information
- Title：游戏大厅 - 狼人杀
- Description：创建房间或加入房间开始对局。
- OG：`og:title`, `og:description`

### 3.2 Layout
- Mobile：单列卡片列表。
- Tablet：房间列表双列栅格（2 columns, gap 12）。
- Desktop：最大宽度 1120px，卡片网格 + 右上角快捷入口。

### 3.3 Page Structure
1. 顶部吸顶栏（Sticky Top Bar）
2. 创建房间区（卡片）
3. 房间列表区（卡片列表/栅格）
4. 底部固定主操作（可选）：移动端在列表很长时提供“创建房间”浮动按钮

### 3.4 Sections & Components
- 顶部栏
  - 左：标题“游戏大厅”
  - 右：按钮“回放”“退出登录”
- 创建房间卡片
  - 输入：房间名（可选）+ 最大人数（下拉/步进器）
  - 主按钮：创建并进入
- 房间列表
  - 房间卡片：房间名、人数（x/y）、状态（等待/进行中/结束）
  - 行为：点击卡片加入（若不可加入则禁用并提示原因）
  - 空态：提示“暂无房间，创建一个开始吧”

## 4. 页面：游戏房间/对局页（/room/:roomId）
### 4.1 Meta Information
- Title：房间 - 狼人杀
- Description：实时对局页面。
- OG：`og:title`, `og:description`

### 4.2 Layout
- Mobile：
  - 顶部：阶段条/倒计时（Sticky）
  - 中部：核心交互区（随阶段切换内容）
  - 底部：主操作条（固定，随阶段变化：准备/发言/投票/夜晚动作）
- Desktop：在不改变信息架构的情况下做左右分栏：左（玩家列表/阶段日志），右（主交互）。

### 4.3 Page Structure
A. 等待区（status=waiting）
1. 房间信息条：房间名、房主标记、人数
2. 玩家列表：席位卡片
3. 操作区：准备/取消准备；房主“开始游戏”“解散房间”

B. 对局中（status=playing）
1. 阶段条：当前阶段 + 倒计时
2. 公共信息区：公告/系统事件（轻量时间线）
3. 核心交互区（按阶段）
4. 底部操作条（固定）

C. 结算（status=ended）
1. 结果卡片：胜负与摘要
2. 行为：返回大厅 / 查看回放

### 4.4 Sections & Components
- 玩家列表（等待区与对局共用）
  - 席位卡：昵称、状态（准备/存活/淘汰）、房主标记
- 阶段条（对局中）
  - 文案：如“夜晚行动”“白天发言”“白天投票”
  - 倒计时：数字 + 进度条
- 发言区（白天发言）
  - 文本输入（多行）+ 发送按钮（最小可用亦可简化为单行）
  - 消息列表：按时间倒序/正序，支持自动滚动到底部
- 投票区（白天投票）
  - 可投目标列表（玩家卡可点选）
  - 主按钮：确认投票（确认前可撤销）
- 夜晚动作区
  - 提示文案（仅本人可见）
  - 目标选择（单选列表）+ 提交按钮
- 系统事件提示
  - Toast：阶段切换/提交成功
  - Banner：关键事件（如“有人出局”）

## 5. 页面：回放页（/replays、/replays/:replayId）
### 5.1 Meta Information
- Title：回放 - 狼人杀
- Description：查看历史对局回放。
- OG：`og:title`, `og:description`

### 5.2 Layout
- Mobile：列表页单列；详情页“播放器区 + 事件时间线”单列滚动。
- Tablet/Desktop：详情页可变为左右分栏（左事件列表、右事件详情/棋盘信息）。

### 5.3 Page Structure
A. 回放列表（/replays）
1. 顶部栏：返回大厅 + 标题
2. 回放卡片列表

B. 回放详情（/replays/:replayId）
1. 顶部栏：返回列表 + 标题
2. 播放控制区：播放/暂停、下一阶段、进度条
3. 事件时间线：按 t（毫秒）排序

### 5.4 Sections & Components
- 回放卡片
  - 信息：时间、房间名（若有）、结果摘要
  - 行为：点击进入详情
- 播放控制
  - Primary：播放/暂停
  - Secondary：下一阶段/关键事件
  - 进度条：可拖动（最小可用可先只展示不可拖动）
- 事件时间线
  - 事件项：时间戳、类型标签、简述
  - 点击事件：展开 payload 关键字段（仅用于回放理解）
